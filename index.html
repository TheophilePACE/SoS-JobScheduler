JobScheduler Architecture

SoS JobScheduler has per default 2 components :

JOC

JobScheduler Operations- Centre (available in sourceforge)

JobScheduler contains a HTTP server which shows job status information in a UI and allows controlling jobs (start and stop jobs, access log files etc.) at run time: Shows the job history and protocols for tasks, orders and the main JobScheduler logs. Provides functions to monitor and control the job scheduling: start jobs, stop jobs, terminate, set parameters, and call up protocols. Read more on this feature in the HTTP server documentation. Jobscheduler writes all this information in a database

JOE

JobScheduler Object Editor (available in sourceforge)

JOE is used to create, modify and delete all JobScheduler objects. JOE is operating directly on a live folder. JOE reduces the complexity of directly accessing the XML structure and validates the configuration against an XML schema. Moreover, it is far easier to manage complex run time job configurations by mouse click. JOE provides wizards for the configuration of jobs and job chains. Read more on this feature in the configuration editor documentation.

JobScheduler Dashboard

available in this Web site, is a Web Dashboard Application based on Java to follow activities from JobScheduler. Dashboard reads this information only in database filled by JOC and don't add charge to JobScheduler Runtime

Install JobScheduler Dashboard

JobScheduler Dashboard is a java application. You need a Java runtime environment to run JobScheduler Dashboard. You can use Oracle Java Standard Edition or OpenJDK but version 1.7 is required. After downloading jar intaller, you can luanch installer with this command :

      java -jar jobscheduler-installer-0.1.0.jar

Installer for JobScheduler install 3 files :
  • The complete application available in the file dashboard.war. This file contains libs and an embedded tomcat launched on port 7000 by default
  • A script to launch jobscheduler dashboard
  • A configuration file "application-jobscheduler.yml" in a subdirectory config inside your installation path
